1.指针地址和指针类型

  • Go中指针操作较为简单，主要由两个符号

  • &取地址，*根据地址取值

  • 取地址:取变量的指针（即在内存的地址）语法如下：

  • 指针取值:对指针使用*操作，就是指针取值，代码如下：

  • ```go
    func main() {
    //指针取值
    a := 10
    b := &a // 取变量a的地址，将指针保存到b中
    fmt.Printf(“type of b:%T\n”, b)
    c := *b // 指针取值（根据指针去内存取值）
    fmt.Printf(“type of c:%T\n”, c)
    fmt.Printf(“value of c:%v\n”, c)
    }

    总结

    • 对变量使用&取地址操作，可以获得变量的地址，即指针变量

    • 指针变量的值是指针地址，即某一个变量的地址

    • 对指针变量进行取值*操作，可以获得指针变量指向的变量的值

  • 网上找到的解答：因为for range创建了每个元素的副本，而不是直接返回每个元素的引用，如果使用该值变量的地址作为指向每个元素的指针，就会导致错误，在迭代时，返回的变量是一个迭代过程中根据切片依次赋值的新变量，所以值的地址总是相同的，导致结果不如预期。

  • 问题的解答！！！！如下面代码：

  • ```go
    package main

    import “fmt”

    type student struct {
    name string
    age int
    }

    func main() {
    m := make(map[string]*student)
    stus := []student{
    {name: “小王子”, age: 18},
    {name: “娜扎”, age: 23},
    {name: “大王八”, age: 9000},
    }

    for index, stu := range stus {
    fmt.Printf(“原来的地址%p”, &stus[index])
    fmt.Println()
    fmt.Printf(“现在的地址%p”, &stu)
    fmt.Println()
    m[stu.name] = &stu
    }
    for k, v := range m {
    fmt.Println(k, “=>”, v.name)
    }
    fmt.Println(m)
    }

    重点: 可以地看到其实原来的地址才是真正的切片中每一个元素的地址，而for range循环结构中的stu的地址却总是固定的，这是因为for range并不是对切片元素的引用，而是每次遍历到哪个元素就为那个元素创建临时变量，并且总是存在相同的地址中，在上面的例子中，就是遍历到”小王子“，就将该结构体存到临时变量stu中，下一次遍历到”娜扎“，就将”娜扎“这个结构体存到stu中，因此在map存储值的时候，&stu总是相同的，因为这个临时变量用来存储每次遍历的元素。所以stu最后保留的就是切片最后一个元素的内容，因为遍历就是如此从开头到结尾的！

  • ```go

    p1.Dream()

  • 方法一res每次值只变化一次，速度过慢。
  • 求a中能最多包括多少个b，由于整数限制原因，无论a，b取何值（变为正数后），a中最多包含230个b(b=1时)
