1.3 自定义类型和类型别名的区别

  • 通过一下代码演示：

  • ```go
    //类型定义
    type NewInt int

    //类型别名
    type MyInt = int

    func main() {
    var a NewInt
    var b MyInt

    fmt.Printf(“type of a:%T\n”, a) //type of a:main.NewInt
    fmt.Printf(“type of b:%T\n”, b) //type of b:int
    }

    总结

      - **类型定义后是真的存在这样的类型，而类型别名在编译完成时并不会出现，出现的是该类型别名的真实名字**

  • 网上找到的解答：因为for range创建了每个元素的副本，而不是直接返回每个元素的引用，如果使用该值变量的地址作为指向每个元素的指针，就会导致错误，在迭代时，返回的变量是一个迭代过程中根据切片依次赋值的新变量，所以值的地址总是相同的，导致结果不如预期。

  • 问题的解答！！！！如下面代码：

  • ```go
    package main

    import “fmt”

    type student struct {
    name string
    age int
    }

    func main() {
    m := make(map[string]*student)
    stus := []student{
    {name: “小王子”, age: 18},
    {name: “娜扎”, age: 23},
    {name: “大王八”, age: 9000},
    }

    for index, stu := range stus {
    fmt.Printf(“原来的地址%p”, &stus[index])
    fmt.Println()
    fmt.Printf(“现在的地址%p”, &stu)
    fmt.Println()
    m[stu.name] = &stu
    }
    for k, v := range m {
    fmt.Println(k, “=>”, v.name)
    }
    fmt.Println(m)
    }

    重点: 可以地看到其实原来的地址才是真正的切片中每一个元素的地址，而for range循环结构中的stu的地址却总是固定的，这是因为for range并不是对切片元素的引用，而是每次遍历到哪个元素就为那个元素创建临时变量，并且总是存在相同的地址中，在上面的例子中，就是遍历到”小王子“，就将该结构体存到临时变量stu中，下一次遍历到”娜扎“，就将”娜扎“这个结构体存到stu中，因此在map存储值的时候，&stu总是相同的，因为这个临时变量用来存储每次遍历的元素。所以stu最后保留的就是切片最后一个元素的内容，因为遍历就是如此从开头到结尾的！

  • 定义格式如下：

    函数体
    }

    可以看到方法属于特定的类型，上面代码中，该方法就属于Person类型的

5.1指针类型的接收者

  • ```go
    // SetAge 设置p的年龄
    // 使用指针接收者
    func (p *Person) SetAge(newAge int8) {
    p.age = newAge
    } 可以看到，通过指针类型的接收者方法，可以修改结构体中的变量值

5.2 值类型的接收者

  • 当方法作用于值类型接收者时，Go语言会在代码运行时将接收者的值复制一份。在值类型接收者的方法中可以获取接收者的成员值，但修改操作只是针对副本，无法修改接收者变量本身。

  • ```go
    // SetAge2 设置p的年龄
    // 使用值接收者
    func (p Person) SetAge2(newAge int8) {
    p.age = newAge
    }

    func main() {
    p1 := NewPerson(“小王子”, 25)
    p1.Dream()
    fmt.Println(p1.age) // 25
    p1.SetAge2(30) // (*p1).SetAge2(30)
    fmt.Println(p1.age) // 25
    }

7.结构体匿名字段

  • 结构体允许成员字段在声明时没有字段名，只有类型，这种没有名字的字段就称为匿名字段

**注意:**里匿名字段的说法并不代表没有字段名，而是默认会采用类型名作为字段名，结构体要求字段名称必须唯一，因此一个结构体中同种类型的匿名字段只能有一个。
